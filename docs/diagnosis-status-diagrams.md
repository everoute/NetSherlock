# NetSherlock 诊断状态 - 图表参考

## 状态机完整图表

```
╔════════════════════════════════════════════════════════════════════════╗
║                         诊断生命周期状态机                             ║
╚════════════════════════════════════════════════════════════════════════╝

                          ┌─────────────┐
                          │   PENDING   │ ← 初始状态
                          │  (等待执行)  │    - 任务在队列中
                          └──────┬──────┘    - 可取消
                                 │ (开始执行)
                                 ↓
                          ┌─────────────┐
                          │   RUNNING   │ ← 活跃状态
                          │  (正在执行)  │    - 不可取消
                          └──┬──┬──┬────┘    - 自主/交互都支持
                             │  │  │
                   ┌─────────┘  │  └──────────┬─────────┐
                   ↓            ↓             ↓         ↓
            ┌──────────────┐ ┌───────┐ ┌────────────┐ ┌────────────┐
            │   WAITING    │ │ ERROR │ │ COMPLETED  │ │ INTERRUPTED│
            │ (等待输入)   │ │(执行  │ │   (成功)   │ │  (已中断)  │
            │             │ │ 错误)  │ │            │ │  (Ctrl+C)  │
            └──┬─────────┬─┘ └───────┘ └────────────┘ └────────────┘
               │         │
           ✅可取消  仅交互模式
               │
         ┌─────┴──────┐
         ↓            ↓
    ┌────────┐   ┌──────────┐
    │ RUNNING│   │ CANCELLED│ (已取消)
    │(继续)  │   │  (用户   │
    │        │   │   取消)  │
    └────────┘   └──────────┘

═══════════════════════════════════════════════════════════════════════

🔴 终止状态 (不可再转换):
   - COMPLETED (成功完成)
   - CANCELLED (用户取消)
   - ERROR (执行错误)
   - INTERRUPTED (被中断)

🟡 活跃状态 (可继续转换):
   - PENDING (等待执行)
   - RUNNING (正在执行)
   - WAITING (等待输入，仅交互模式)
```

┌─────────────────────────────────────────────────────────────┐
│ pending          ⏰ Clock    灰色    中性      -             │

│ completed        ✅ CheckCircle 绿色  成功     -             │
├─────────────────────────────────────────────────────────────┤
│ cancelled        ❌ XCircle    灰色    取消     -             │
├─────────────────────────────────────────────────────────────┤
│ error            ⚠️ AlertCircle 红色  错误     -             │
├─────────────────────────────────────────────────────────────┤

═══════════════════════════════════════════════════════════════

颜色类别:
  🟩 成功:  绿色 (completed)
  🟥 错误:  红色 (error) 
  🟨 等待:  黄色 (waiting)
  🟧 中断:  橙色 (interrupted)

## 操作按钮流程

```
╔════════════════════════════════════════════════════════════╗
║            状态 → UI 按钮 → 用户操作 流程                  ║
╚════════════════════════════════════════════════════════════╝

pending
    │
    └─→ [❌ Cancel]
            │
            └─→ POST /diagnose/{id}/cancel
                  │
                  └─→ 状态变为 CANCELLED

running
    │
    └─→ [👁️ Logs]
            │
            └─→ 导航到 /tasks/{id}

waiting
    │
    └─→ [👁️ Logs]
            │
            └─→ 导航到 /tasks/{id}

completed
    │
    └─→ [📄 Report]
            │
            └─→ 导航到 /reports/{id}

error
    │
    └─→ [👁️ Logs]
            │
            └─→ 导航到 /tasks/{id}

interrupted
    │
    └─→ [👁️ Logs]
            │
            └─→ 导航到 /tasks/{id}

cancelled
    │
    └─→ [🔄 Retry]
            │
            └─→ 提交新诊断请求
```

## 交互模式 vs 自主模式

```
╔════════════════════════════════════════════════════════════╗
║           模式对比 - 状态流程不同                          ║
╚════════════════════════════════════════════════════════════╝

自主模式 (AUTONOMOUS)
═══════════════════════════════════════════════════════════

  PENDING  →  RUNNING  →  COMPLETED ✓
                   ↓
                 ERROR (异常时)
                   ↓
              INTERRUPTED (Ctrl+C)

  特点:
  - 无中途暂停
  - 不使用 WAITING 状态
  - 自动执行到底


交互模式 (INTERACTIVE)
═══════════════════════════════════════════════════════════

  PENDING  →  RUNNING  ⟿  COMPLETED ✓
                  ↓
              WAITING ← 用户确认或取消
                  ↓
         [用户选择]
         ↙          ↘
   继续执行        取消
      ↓              ↓
    RUNNING      CANCELLED
      ↓              ↓
  COMPLETED ✓    [结束]

  特点:
  - 在检查点暂停
  - 使用 WAITING 状态
  - 等待用户决策
```

## 网络请求序列图

```
╔════════════════════════════════════════════════════════════╗
║              前端 ↔ 后端 状态流通序列                       ║
╚════════════════════════════════════════════════════════════╝

前端                                    后端
 │                                       │
 │  1. POST /diagnose                    │
 │──────────────────────────────────────→│
 │  (创建诊断)                            │
 │                                 status: PENDING
 │  ← DiagnosisResponse                  │
 │←──────────────────────────────────────│
 │
 │  2. GET /diagnose/{id} (轮询 2~5秒)   │
 │──────────────────────────────────────→│
 │                                  status: RUNNING
 │  ← DiagnosisResponse              或 WAITING
 │←──────────────────────────────────────│
 │
 │  [可选] 3. POST /diagnose/{id}/cancel │
 │──────────────────────────────────────→│
 │  (仅 PENDING/WAITING 状态)             │
 │                               status: CANCELLED
 │  ← 取消成功响应                        │
 │←──────────────────────────────────────│
 │
 │  4. GET /diagnose/{id}/report (COMPLETED后)
 │──────────────────────────────────────→│
 │                              status: COMPLETED
 │  ← 诊断报告 (Markdown)                 │
 │←──────────────────────────────────────│
 │

轮询停止条件:
  - status ∈ {COMPLETED, CANCELLED, ERROR, INTERRUPTED}
  - 或 5 次轮询失败
```

## API 状态转换时间线

```
╔════════════════════════════════════════════════════════════╗
║         API 请求 → 状态转换 → 响应 时间线                   ║
╚════════════════════════════════════════════════════════════╝

时间轴
──────────────────────────────────────────────────────────────

T0    POST /diagnose
      └─→ 创建诊断
          status: PENDING
          started_at: NULL

T0+ε  worker 获取任务
      └─→ 状态转换
          status: RUNNING
          started_at: NOW

T0+x  [交互模式] 到达检查点
      └─→ 状态转换
          status: WAITING

T0+y  [交互模式] 用户确认
      └─→ POST /diagnose/{id}/confirm
          status: RUNNING (再次)

T0+z  诊断完成
      └─→ 状态转换
          status: COMPLETED/ERROR/INTERRUPTED
          completed_at: NOW

T0+z+ε  前端获取报告
        └─→ GET /diagnose/{id}/report
            └─→ markdown_report: "# 报告..."
```

## 错误处理流程

```
╔════════════════════════════════════════════════════════════╗
║           错误场景 → 状态转换 → 错误响应                    ║
╚════════════════════════════════════════════════════════════╝

取消错误 (Cannot Cancel)
═════════════════════════════════════

  状态: RUNNING
    │
    ├─→ POST /diagnose/{id}/cancel
    │
    └─→ ❌ 400 Bad Request
        {
          "detail": "Cannot cancel diagnosis in running status"
        }

  可取消: PENDING, WAITING
  不可取消: RUNNING, COMPLETED, ERROR, INTERRUPTED, CANCELLED


诊断不存在 (Not Found)
════════════════════════

  POST /diagnose/{id}/cancel
    │
    └─→ ID 不在存储中
        │
        └─→ ❌ 404 Not Found
            {
              "detail": "Diagnosis {id} not found"
            }


执行错误 (Execution Error)
═════════════════════════

  状态: RUNNING
    │
    └─→ 执行异常 (网络、权限、超时等)
        │
        └─→ 状态转换: ERROR
            completed_at: NOW
            error: "详细错误信息"
            │
            └─→ API 响应
                {
                  "status": "error",
                  "error": "...",
                  "summary": "..."
                }


中断 (Interrupted)
══════════════════

  状态: RUNNING
    │
    └─→ Ctrl+C / 信号处理
        │
        └─→ 捕获 KeyboardInterrupt
            │
            └─→ 状态转换: INTERRUPTED
                completed_at: NOW
                │
                └─→ 清理资源
                    正常退出
```

## 状态可达性图

```
╔════════════════════════════════════════════════════════════╗
║         从任意状态 可以到达 哪些状态                         ║
╚════════════════════════════════════════════════════════════╝

        ┌─────────┬─────────┬─────────┬──────────┬──────────┐
        │从 \ 到  │ RUNNING │ WAITING │COMPLETED │  ERROR   │
├────────┼─────────┼─────────┼─────────┼──────────┼──────────┤
│PENDING │   ✅    │   ❌    │   ❌    │   ❌     │   ❌     │
├────────┼─────────┼─────────┼─────────┼──────────┼──────────┤
│RUNNING │   -     │   ✅    │   ✅    │   ✅     │   ✅     │
├────────┼─────────┼─────────┼─────────┼──────────┼──────────┤
│WAITING │   ✅    │   -     │   ❌    │   ❌     │   ❌     │
├────────┼─────────┼─────────┼─────────┼──────────┼──────────┤
│其他    │   ❌    │   ❌    │   ❌    │   ❌     │   ❌     │
└────────┴─────────┴─────────┴─────────┴──────────┴──────────┘

说明:
  ✅ = 可以转换
  ❌ = 不可转换
  -  = 同一状态
```
